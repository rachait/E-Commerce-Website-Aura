# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately.
Do not create public GitHub issues for security-sensitive reports.

Include:

- A clear description of the issue
- Reproduction steps
- Affected components
- Suggested mitigation (if known)

## Response Process

- We will acknowledge receipt as soon as possible.
- We will validate and triage severity.
- We will provide patch status updates.

## Scope

Security-sensitive areas include:

- Authentication and authorization
- Payment flow integrations
- Secrets and environment configuration
- API rate limiting and abuse prevention
